🔴 Infrastructure Spec added 2026-03-02

Real-Time Portal — No More Refreshing

Every status change currently requires a page refresh. The portal should update instantly — like Slack or Linear. Root cause: JSON files as a database + cron-driven data.json regeneration pipeline. Three options evaluated: WebSockets (overkill), Server-Sent Events (DIY option), and Supabase Realtime (recommended — solves real-time AND the flat-file problem in one move).

Recommended: Supabase Realtime

Managed Postgres + built-in real-time subscriptions. Change a row → every browser sees it instantly. Free tier. Replaces content-queue.json, content-schedule.json, daily-goals.json with real DB tables. Agents write rows; portal subscribes live. No more data.json generation or GitHub deploy cycle for content state.

Migration Path
1. Opus specs full schema (queue, schedule, goals, agent_log)
2. Migrate existing JSON → Supabase
3. Rewrite portal with Supabase JS client + real-time subscriptions
4. Update all agents to write to Supabase instead of JSON files
5. Retire data.json pipeline entirely

🔴 Infrastructure Spec added 2026-03-02

Replace JSON File Pipeline with Real Database + API

The current content pipeline uses flat JSON files as its database. Every status change requires: edit JSON → run generate-data.js → git commit + push → wait for Cloudflare deploy → refresh browser. That's 3–4 minutes for what should be instant. It's fragile, has no audit trail, and concurrent writes can corrupt the file.

Proposed Architecture
📦 Database

SQLite on Mac Mini (or Cloudflare D1). Single source of truth for all content items, statuses, scheduled times, tweet IDs, audit history.

🔌 API Layer

Node/Express on Mac Mini (port 3001), exposed via Cloudflare Tunnel. Endpoints: GET /queue, PATCH /items/:id/status, GET /published, POST /items.

🖥️ Portal

Stays static HTML. Swap data.json fetches for live API calls. Status changes write directly to DB — portal reflects instantly. No deploy cycle for content state changes.

🤖 Agents

Samantha writes drafts to DB via API. Publish cron reads approved items from DB. All agents share the same API — no file conflicts.

📋 Audit Trail

Every status change logged with timestamp, actor, and reason. Full history — no more "how did this get back in the queue?"
