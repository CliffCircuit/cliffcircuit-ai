🔴 Infrastructure Spec added 2026-03-02

Real-Time Portal — No More Refreshing

Every status change currently requires a page refresh. The portal should update instantly — like Slack or Linear.

Recommended: Supabase Realtime

Managed Postgres + built-in real-time subscriptions. Change a row → every browser sees it instantly. Free tier. Replaces content-queue.json, content-schedule.json, daily-goals.json with real DB tables.

Migration Path
1. Opus specs full schema (queue, schedule, goals, agent_log)
2. Migrate existing JSON → Supabase
3. Rewrite portal with Supabase JS client + real-time subscriptions
4. Update all agents to write to Supabase instead of JSON files
5. Retire data.json pipeline entirely

🔴 Infrastructure Spec added 2026-03-02

Replace JSON File Pipeline with Real Database + API

The current content pipeline uses flat JSON files as its database. Every status change requires: edit JSON → run generate-data.js → git commit + push → wait for deploy → refresh.

Proposed Architecture
📦 Database

SQLite on Mac Mini (or Cloudflare D1). Single source of truth.

🔌 API Layer

Node/Express on Mac Mini (port 3001), exposed via Cloudflare Tunnel.

🖥️ Portal

Stays static HTML. Swap data.json fetches for live API calls.

🤖 Agents

All agents share the same API — no file conflicts.
